# 🛒 Q-Commerce Hyper-Local Demand Forecasting Engine

A Streamlit dashboard that forecasts **hourly, SKU-level demand** for Q-commerce
dark stores (think Zepto / Blinkit / Swiggy Instamart) using a full ML pipeline —
from synthetic data generation through descriptive & diagnostic analysis to
Random Forest, XGBoost, LightGBM, and Prophet models.

---

## 📂 Project Structure

```
qcommerce_forecast/
├── app.py                  ← Main Streamlit dashboard (5 pages)
├── generate_data.py        ← Synthetic dataset generator (standalone)
├── feature_engineering.py  ← Feature engineering utilities (standalone)
├── synthetic_data.csv      ← Pre-generated dataset (~675 rows)
├── requirements.txt        ← Python dependencies
├── README.md               ← This file
└── .streamlit/
    └── config.toml         ← UI theme
```

## 🚀 Quick Start (Local)

```bash
# 1. Clone / download the repo
git clone https://github.com/<your-username>/qcommerce-forecast.git
cd qcommerce-forecast

# 2. Create a virtual environment (recommended)
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Run the dashboard
streamlit run app.py
```

The app opens at **http://localhost:8501** in your browser.

> **Note:** `synthetic_data.csv` is bundled in the repo.  
> If you delete it, the app auto-regenerates it on first run.

---

## ☁️ Deploy on Streamlit Cloud (GitHub)

1. Push this entire folder to a **public GitHub repository**.
2. Go to [share.streamlit.io](https://share.streamlit.io) → **New app**.
3. Select your repo, set **Main file path** to `app.py`.
4. Click **Deploy** — Streamlit Cloud reads `requirements.txt` automatically.

> **Tip:** Prophet installation on Streamlit Cloud can take 3–5 minutes on
> first deploy. Subsequent cold-starts are faster.

## 🧪 Synthetic Dataset

Generated by `generate_data.py` — **675 rows** representing:

| Dimension | Values |
|-----------|--------|
| Dark Stores | DS001_Andheri_MUM · DS002_Koramangala_BLR · DS003_CP_DEL |
| SKUs | SKU001_Milk_1L · SKU002_Eggs_12pk · SKU003_Bread_500g |
| Period | 1 Apr 2024 → 25 Apr 2024 |
| Granularity | 3 peak hours/day: 09:00 · 18:00 · 21:00 |

**Demand patterns baked in:**

- 🌧️ Rain → +25–40 % uplift per SKU (comfort-buying)
- 🎉 Weekends → +45 % vs weekdays
- 🏖️ Public holidays → +65 % spike
- 🎪 Local events (IPL) → +28 % uplift
- 🌡️ Extreme heat → −10–12 % suppression
- ⏰ Evening slot (18:00) → highest demand across all conditions

## 🤖 Models

| Model | Type | Notes |
|-------|------|-------|
| **Random Forest** | Tree ensemble | 200 trees, max_depth=12 |
| **XGBoost** | Gradient boosting | 200 rounds, lr=0.05, subsample=0.8 |
| **LightGBM** | Fast gradient boosting | 200 rounds, num_leaves=31 |
| **Prophet** | Additive time-series | Trained on aggregated platform demand |

**Train / Test split:** 80 % / 20 % — sorted by timestamp (no data leakage).

## ⚙️ Feature Engineering (33 features)

```
Temporal   : hour, day_of_week, day_of_month, month, week_of_year, is_weekend
Cyclical   : hour_sin/cos, dow_sin/cos
Lag        : lag_1, lag_2, lag_3, lag_7
Rolling    : mean/std/max over 3 & 7 periods
Exp. MA    : ema_3, ema_7
Exogenous  : temperature_celsius, weather_encoded, rain_flag, heat_flag
Interaction: rain_evening, weekend_evening, holiday_evening
Flags      : public_holiday_flag, local_event_flag, stockout_flag
Entity     : store_encoded, sku_encoded
```
